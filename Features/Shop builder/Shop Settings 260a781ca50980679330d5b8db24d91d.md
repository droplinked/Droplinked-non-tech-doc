# Shop Settings

## **Shop Settings Features Summary**

Droplinked provides a comprehensive **Shop Settings** system that allows merchants to fully customize and manage their storefronts. These features include design customization, payment and wallet integration, login method configuration, API access, age gating, address management, default product and currency settings, post-purchase behavior, and user management. Merchants can configure their shops to create a tailored experience for customers while maintaining control over operational and financial settings.

---

## **Shop Settings Features Overview**

### 1. **Create Shop**

- **Description**: Shops are automatically created when a merchant registers on Droplinked. No manual steps are required to initialize a shop, ensuring that every merchant has a functional storefront from the start.

### 2. **Design**

- **Description**: Merchants can fully customize their storefront, including Product Listing Pages (PLP), banners, themes, and layout using the Droplinked Design Maker. This allows for complete branding and presentation control.

### 3. **Settings**

- **Description**: Merchants can manage all shop settings in one place, including:
    - **Payment Settings**: Select which methods customers can use (Stripe, Coinbase, Web3 wallets) and define wallet addresses and split percentages.
    - **Login Method Settings**: Configure how customers can log in and create profiles (Google, MetaMask, Unstoppable, Phantom Wallet).
    - **API Keys**: Access Droplinked APIs with a unique merchant API key.
    - **Connect Wallet**: Manage connected wallets and assign shares for sales.
    - **Age Gating**: Enable modal verification for visitors under 18.
    - **Default Product**: Configure a default product for the shop.
    - **Address Management**: Add or edit shop addresses.
    - **Post-Purchase Settings**: Configure post-order behaviors and notifications.
    - **Default Currency**: Set shop currency for product and cart pricing.
    - **User Management**: Manage customer accounts, roles, and access permissions.

### 4. **Payment Methods**

- **Description**: Merchants can select which payment options their customers can use for purchases, including Stripe, Coinbase, or wallet payments. Merchants must enable at least one payment method. Payments made via Stripe or Coinbase will charge the customer and deposit the merchantâ€™s share directly to their connected account. Wallet payments can distribute revenue to one or multiple wallet addresses, or to internal Droplinked credit if no wallet is provided.

### 5. **Login Methods**

- **Description**: Merchants can configure available login methods for customers. At least one login method must be enabled. Supported options include Google, MetaMask, Unstoppable Wallet, and Phantom Wallet, allowing for flexible authentication and user profile creation.

### 6. **API Keys**

- **Description**: Merchants can use Droplinked API keys to integrate platform functionality into external systems or custom code, enabling programmatic access to shop data and operations.

### 7. **Age Gating**

- **Description**: Merchants can enable an age verification modal that requires visitors to confirm they are over 18 before accessing the shop, ensuring compliance with content restrictions.

### 8. **Address Management**

- **Description**: Merchants can add, edit, and manage shop addresses used for shipping, billing, and operational purposes.

### 9. **Post-Purchase**

- **Description**: Merchants can configure post-purchase behavior, such as order confirmation, follow-up notifications, and loyalty or reward actions.

### 10. **Default Product & Currency**

- **Description**: Merchants can set a default product for their shop and define the default currency for pricing and checkout, ensuring consistency across the storefront.

### 11. **User Management**

- **Description**: Merchants can manage customer accounts, assign roles, and control access to shop features, enabling efficient customer relationship management and shop administration.
